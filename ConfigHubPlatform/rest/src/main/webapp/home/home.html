How it Works

Configuring distributed systems can be complex and difficult to manage.  Traditional property files fail to address some fundamental needs, while other solutions require custom implementation and maintenance.   ConfigHub simplifies developer workflow, reduces errors and provides greater software security.

  • Centralized Configuration

    Configuration for all applications is maintained in your private ConfigHub repository. Organize users in teams, and control read/write per keys or context.

    Manage configuration, tag releases, view revisions and compare configuration between contexts, times and tags.

  • ConfigHub Solution Hosting

    Our hosted solution is the simplest and fastest way to get started. Our data-centers are in the US, Europe and Australia.

    Enterprise release can be hosted in your cloud, server or a VM. Looking for a private environment, get in touch.

  • Pulling

    To pull configuration, applications provide a token and their context through a ConfigHub API, or our RESTful service.

    The APIs provide functionality for type-checking, decryption, config backup/read on local disk and fetching config from specific time or tag. All API requests are tracked, and you can always see which application, from which source requested resources.

Pulling Configuration

There are many ways for an application to receive configuration maintained in a ConfigHub repository. Which path you choose depends on your specific needs.

ConfigHub API is provided over HTTPS RESTful service in JSON format, making it easy to customize. Several language specific APIs are available and ready to plug into your applications.

One of the common alternatives to the API is staging, where configuration is pushed on the target hosts to avoid any code modification.
